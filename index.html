📍 Meta Tag

  • [ Viewport ]

  • width=device-width : 페이지를 자동으로 사용자 디바이스 크기에 맞춤
  • initial-scale=1.0 : 페이지 로딩 시의 확대 비율
  • maximum-scale=1.0 : 허용가능한 확대비율의 최대치
  • minimum-scale=1.0 : 허용가능한 축소비율의 최대치
  • user-scalable=0 : 사용자 확대보기 유무 (yes==1, no==0)

📍 Touch Event

  • 모바일 디바이스 브라우저에서는 touch 이벤트를 사용

    touchstart => touchmove => touchend (mousedown => mousemove => mouseup)

  • 이벤트 등록은 다음과 같이 PC브라우저에서 적용한 방식과 동일함
  • touch EventListener에 전달되는 event객체에는 target 속성과
    touches와 changedTouches 라는 하위 속성 내제
  • touches와 changedTouches

    두 개의 값은 배열타입. 그 이유는 하나뿐인 마우스 포인터만 있는 PC와 다르게,
    모바일 기기에서는 1개 이상의 터치영역이 생기기 때문. (손가락이 여러개이므로) 때문에 어떤 손가락은 touchstart이고 어떤 손가락은 touchmove중인 상태가 발생 된다.

  • touches

    : 모든 touch 이벤트

  • changedTouches

    : 이전대비 현재 변화가 일어난 모든 이벤트

  • [ touchstart 와 touchend 에서 ] changedTouches 는 언제나 1개만 들어옴
  • [ changedTouches 가 여러 개 들어오는 상황은 ] touchmove 에서 2개 이상의 손가락을 동시에 움직일 때

    (index는 그냥 index일 뿐 처음 터치한 손가락이 뭔지, 두 번째 터치한 손가락이 뭔지를 알려주지는 않음.)

☑️ touch move 주의할점

touchmove 는 touchstart 나 touchend 와 달리 일반적으로 브라우저가 처음부터 소유함.
따라서 최초 한 번은 발생하지만 그 이후는 브라우저의 스크롤이 가져가 버리게됌.

만약 어떤 dom 에서 touchmove 이벤트를 사용하겠다는 뜻은 바꿔 말하면 그 영역에서는 스크롤이 안일어나게 하겠다 는 의미.

따라서 touchmove 의 경우는 반드시 e.preventDefault() 를 호출하여 막아주지 않으면 제대로 작동 X
